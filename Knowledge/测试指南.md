## 测试方案二：对接真实知识库系统测试
需要能连接到`knowledge-base-system`的PostgreSQL数据库。

### 步骤1：配置数据库连接
在`config.py`中修改知识库系统的PG配置：
```python
KBS_PG_HOST = "你的知识库系统PG地址"
KBS_PG_PORT = 5432
KBS_PG_DATABASE = "knowledge_base"  # 按实际数据库名修改
KBS_PG_USER = "postgres"  # 按实际用户名修改
KBS_PG_PASSWORD = "你的PG密码"
```

### 步骤2：测试数据库连接
创建一个简单的测试脚本`test_pg_connection.py`：
```python
from Knowledge.importer import KBSImporter

importer = KBSImporter()
chunks = importer.get_unprocessed_relation_chunks(limit=5)
print(f"获取到 {len(chunks)} 个知识块")
for chunk in chunks:
    print(f"- {chunk['chunk_id']}: {chunk['title']}")
importer.close()
```

运行：
```bash
python test_pg_connection.py
```
如果能正常返回知识块列表，说明连接成功。

### 步骤3：运行全量导入测试
```bash
# 先测试少量数据
python -m Knowledge.import_from_kbs.py --limit 10
```

## 结果验证方法
### 1. 验证知识图谱数据
在Neo4j中执行查询：
```cypher
// 查询所有实体和关系
MATCH (s)-[r:HAS_RELATION]->(o)
RETURN s.name, r.type, o.name, r.source_chunks
LIMIT 20
```
应该能看到从知识块中抽取的三元组，且`source_chunks`字段对应知识库系统的`chunk_id`。

### 2. 验证增量处理
再次运行导入脚本：
```bash
python -m Knowledge.import_from_kbs.py --limit 10
```
预期输出：
```
获取到 0 个待处理知识块
没有需要处理的新知识块
```
说明已处理进度记录功能正常，不会重复处理。

### 3. 验证去重功能
构造两个内容语义相似的知识块，比如：
- 知识块1："用户积分和消费金额关联"
- 知识块2："消费金额与用户积分相关"
运行后应该只会创建两个实体（用户积分、消费金额）和一条关系，不会重复创建。

## 常见问题排查
1. **API调用失败**：检查`DASHSCOPE_API_KEY`和`VOL_API_KEY`是否正确，网络是否能访问火山引擎/阿里云服务
2. **Neo4j连接失败**：检查Neo4j服务是否启动，地址、用户名、密码是否正确
3. **PG连接失败**：检查PG的IP、端口、用户名密码是否正确，PG是否允许外部连接
4. **没有拉取到知识块**：检查知识库系统的`chunks`表中是否有`knowledge_type`为`relation`的记录
